# CountFaith 概念設計書

## プロジェクト概要
CountFaithは、偽造医薬品を専門家の目で防ぐ、医薬品流通監査システムです。AIが検知したリスク地点に、地域の潜在薬剤師を「監査官（Supervisor）」として派遣し、真贋判定と管理体制をチェックします。

## 主要機能

### 1. リスク拠点の可視化
- グローバルマップでリスク拠点を表示
- リスクレベル別の色分け（高・中・低）
- 監査状況の統計表示

### 2. 薬剤師検索（Crustdata統合）
- 拠点周辺の薬剤師を自動検索
- 地理的距離フィルター（50km以内）
- 経験年数・ライセンス情報の表示
- 実際のLinkedInプロフィールから取得

### 3. 監査クエスト作成
- リスク拠点の選択
- 周辺薬剤師の自動検出
- 薬剤師の選択と割当
- Proof of Faith（真実証明）の発行

### 4. 薬剤師管理
- 登録薬剤師の一覧表示
- 稼働状況の可視化
- 実績管理

## 三者のエコシステム

### 政府・製薬会社（依頼主）
- 監査の実行指示
- 報酬の支払い
- 流通の信頼性確保

### 潜在薬剤師（監査官）
- 流通拠点での真贋判定
- 管理体制のチェック
- 専門資格の活用
- 柔軟な働き方での高額報酬

### 流通拠点（被監査者）
- 薬剤師の入室許可
- データ照合の協力
- CountFaith認定による信頼獲得

## ユーザーフロー

```
ダッシュボード
  ↓
監査作成 → 拠点選択 → 薬剤師検索（Crustdata） → 薬剤師選択 → クエスト割当（Blaxel） → Proof発行
  ↓
薬剤師管理 → 稼働状況確認
```

## 技術スタック
- Frontend: Next.js 15, React, TypeScript, Tailwind CSS
- APIs:
  - **Crustdata**: 薬剤師データベース検索（地理的検索）
  - **Blaxel**: 最適割当とProof of Faith発行
- Map: Leaflet
- UI: shadcn/ui
